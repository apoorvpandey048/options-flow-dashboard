# Render.com Deployment Guide

## Quick Setup (5-10 minutes)

### Step 1: Create Render Account
1. Go to https://render.com
2. Sign up with GitHub (recommended)
3. Authorize Render to access your repositories

### Step 2: Deploy from GitHub
1. Click **"New +"** button (top right)
2. Select **"Blueprint"**
3. Connect your GitHub account if not already connected
4. Select repository: `apoorvpandey048/options-flow-dashboard`
5. Click **"Apply"**

### Step 3: Configure Environment Variables
Render will automatically create 2 services. Configure them:

#### Backend Service:
1. Go to **options-dashboard-backend** service
2. Click **"Environment"** tab
3. Add these variables:
   ```
   JWT_SECRET_KEY=<auto-generated by Render>
   FLASK_ENV=production
   DATA_PROVIDER=simulated
   ALLOWED_ORIGINS=<your-frontend-url>
   ```
4. **After frontend deploys**, update `ALLOWED_ORIGINS` with frontend URL

#### Frontend Service:
1. Go to **options-dashboard-frontend** service
2. Click **"Environment"** tab
3. Add:
   ```
   REACT_APP_API_URL=<your-backend-url>
   ```
4. Copy backend URL from backend service dashboard

### Step 4: Get Your URLs
After deployment completes (5-10 min):
- Backend: `https://options-dashboard-backend.onrender.com`
- Frontend: `https://options-dashboard-frontend.onrender.com`

### Step 5: Update CORS
1. Go back to backend environment variables
2. Update `ALLOWED_ORIGINS` to your frontend URL
3. Save and redeploy

### Step 6: Test the App
1. Visit your frontend URL
2. Register a new account
3. Test the features

## Troubleshooting

### Backend won't start:
- Check logs in Render dashboard
- Verify all environment variables are set
- Ensure `JWT_SECRET_KEY` is not empty

### Frontend can't connect to backend:
- Verify `REACT_APP_API_URL` in frontend env
- Check backend is running (green status)
- Update `ALLOWED_ORIGINS` in backend

### Free tier limitations:
- Backend sleeps after 15 min inactivity
- First request after sleep takes 30-60 seconds
- Upgrade to paid tier ($7/month) for always-on
